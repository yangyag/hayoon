# 화면/콘텐츠 상세 계획서 (v1)

## 1. 사용자 플로우
1. 첫 화면(환영)
- 문구: `하윤이 환영해~`
- 큰 시작 버튼: `시작하기`
- 배경: 밝고 화려한 애니메이션 느낌

2. 책장 화면
- 도서관 책장 스타일 배경
- 책은 1권만 노출: `가나다 한글`
- 책 클릭 시 글자 선택 화면으로 이동

3. 글자 선택 화면
- 범위: `가, 나, 다, 라, 마, 바, 사, 아, 자, 차, 카, 타, 파, 하`
- 각 글자는 큰 카드 버튼으로 제공
- 글자 선택 시 학습 플레이 화면으로 이동

4. 글자 학습 플레이
- 1단계: 선택한 글자 크게 표시 (예: `가`)
- 2단계: `다음` 버튼 클릭 시 해당 글자의 단어/이미지를 랜덤 노출
- 단어는 화면 하단 텍스트로 표시
- 예: `가방`에서 `가`는 굵게 표시
- `다음` 클릭 시 같은 글자의 다른 단어로 진행

## 2. 랜덤 노출 규칙
- 기본 개수: 글자당 5개 단어
- 랜덤 방식: 중복 없는 셔플 순회
- 5개를 모두 보여준 뒤 다시 셔플
- `이전` 버튼은 MVP에서 제외
- `홈으로` 버튼은 항상 제공

## 3. 텍스트 강조 규칙
- 표시 형태: 단어의 첫 글자(선택 글자)만 bold
- 예시:
- 선택 글자 `가` + 단어 `가방` => `**가**방`
- 선택 글자 `나` + 단어 `나비` => `**나**비`

## 4. 화면별 필수 컴포넌트
- WelcomePage
- LibraryPage
- BookCard (`가나다 한글`)
- LetterGrid (가~하)
- LetterIntroCard (큰 글자 1장)
- WordCard (이미지 + 단어 + 다음 버튼)

## 5. 데이터 구조(초안)
- `Book`
- `id`: `ganada-hangul`
- `title`: `가나다 한글`
- `letters`: `Letter[]`

- `Letter`
- `id`: `ga` 등 영문 키
- `label`: `가` 등 한글
- `items`: `WordItem[]`

- `WordItem`
- `id`: 고유 id
- `word`: `가방`
- `imagePath`: 정적 파일 경로

## 6. 콘텐츠 운영 정책
- 초기에 `가~하` 글자를 모두 노출
- 각 글자당 단어 5개를 기본 세트로 사용
- 이미지 파일은 사용자가 직접 채워넣는다.
- 파일이 없으면 기본 placeholder 이미지를 사용한다.

## 7. 접근성/사용성 가이드 (40개월 기준)
- 버튼 최소 높이 44px 이상
- 텍스트 크기 크게 유지(기본 28px 이상)
- 조작 포인트 단순화(주요 버튼 1~2개)
- 문구는 짧고 반복 가능하게 구성
